6  Network Flows

6.1 Model Formulation

6.1.1 Motivation

We want to figure out the maximum amount of some commodity can be transported from a certain node \(s\) to another point \(t\) of a network.

This can take the form of any of the following (among many other examples):

  • Oil, water, really any fluid in pipeline network;
  • Vehicle traffic in a roadway network;
  • Information in a communication network;
  • Spread of disease in pandemic models.

6.1.2 Standard Constraints

Regardless of physical application/interpretation of the network, we have the following standard collection of constraints.

6.1.2.1 Capacity constraints

Each pipe cannot carry more than its capacity. - Let \(x_{ij}\) denote the amount of commodity carried from \(i\) to \(j\) and let \(k_{ij}\) denote the capacity of the pipe from \(i\) to \(j\). - Then we must have \[ 0 \le x_{ij} \le k_{ij}. \]

6.1.2.2 Balance constraints

The commodity cannot enter or leave the network except at source \(s\) and sink \(t\).

  • Consider node \(h\) (not \(s\) or \(t\)).
  • The flow of the commodity into \(h\) must be equal to the flow out of \(h\): \[ \sum_{(i,h) \in \delta^{-}(h)} x_{ih} = \sum_{(h,j) \in \delta^{+}(h)} x_{hj}. \]

6.1.3 The Maximum Flow Problem

Given a directed graph \(G = (V,A)\), arc capacity function \(k: A \mapsto \mathbf{R}_+\), and nodes \(s,t \in V\).

Definition 6.1 (Maximum Flow Problem) Find a function \(x: A \mapsto \mathbf{R}_+\) (called a flow) satisfying:

  • capacity constraints: \[ 0 \le x_{ij} \le k_{ij} \] for all \((i,j) \in A\);

  • balance constraints: \[ \sum_{(i,h) \in \delta^{-}(h)} x_{ih} = \sum_{(h,j) \in \delta^{+}(h)} x_{hj} \] for all $h V {s,t},

  • maximizing the net flow value: \[ \varphi(\{s\}) = \varphi(x) = \sum_{(s,j) \in \delta^{+}(s)} x_{sj} - \sum_{(i,s) \in \delta^{-}(s)} x_{is}. \]

6.1.4 Example

Example 6.1 Figure 6.1 gives an example of a flow \(x\) for the given graph \(G\) with capacity \(k\).

The flow can be written as \[ \begin{aligned} x_{s1} &= 3, &&& x_{s2}&=0 \\ x_{12} &= 2, &&& x_{1t} &= 1 \\ x_{21} &= 0, &&& x_{2t} &= 2. \end{aligned} \] This flow \(x\) has value \(\varphi(x) = 3.\)

6.1.5 Formulation as a Linear Program

We can write the maximum flow problem as the linear program or linear optimization problem1: \[ \begin{array}{ll} \max & \varphi \\ \text{s.t.} & \displaystyle \sum_{(h,j) \in \delta^+(h)} x_{hj} - \sum_{(i,h) \in \delta^-(h)} x_{ih}= \begin{cases} + \varphi, & \text{if } h =s, \\ - \varphi, & \text{if } h = t,\\ 0, &\text{otherwise}, \end{cases} \\ & 0 \le x_{ij} \le k_{ij} \text{ for all } (i,j) \in A, \\ & \varphi \ge 0. \end{array} \]

6.2 Cuts and Flow

6.2.1 \(st\)-cuts

Given a subset of nodes \(S \subseteq V\) with \(s \in S\) and \(t \in V \setminus S\).

Definition 6.2 An st-cut is the set of arcs that enter or leave \(S\): \[ \delta(S) = \delta^+(S) \cup \delta^{-}(S). \]

Definition 6.3 The (forward) capacity of the \(st\)-cut is \[ k(\delta(S)) = \sum_{(i,j) \in \delta^+(S)} k_{ij}. \]

6.2.2 Net Flow across a Cut

Definition 6.4 The net flow across \(\delta (S)\) is \[ \varphi(\delta(S)) := \sum_{(i,j)\in \delta^+(S)} x_{ij} - \sum_{(i,j) \in \delta^{-}(S)} x_{ij}. \]

6.2.3 Example

Example 6.2 Consider the graph given in Figure 6.2.

The forward and backward cuts induced by \(S\) are given in Figure 6.3.

The \(st\)-cut induced by \(S\) is \[ \delta(S) = \delta^{+}(S) \cup \delta^{-}(S) = \{(s,1), (2,1), (2,t), (1,2)\}. \] The forward capacity of this cut is \[ k(\delta(S)) = 7 + 1 + 5 = 13. \] The value of this cut is \[ \varphi(\delta(S)) = 3 + 0 + 2 - 2 = 3. \]

6.2.4 Conservation of Flow Across Cuts

Lemma 6.1 Given a flow \(x\) with net value \(\varphi\), we have \[ \varphi(\delta(S)) = \varphi \] for every \(S \subseteq V\) with \(s \in S\), \(t \in V \setminus S\).

6.13 The Dual Problem: Min Cut

Definition 6.5 (Minimum Cut Problem) Given a directed graph \(G = (V, A)\) with arc capacity function \(k: A \mapsto \mathbf{R}_+\) and two nodes \(s,t \in V\).

Find an \(st\)-cut of minimum capacity.

6.16 Weak Duality Revisited

The relationship \[ \begin{aligned} & \varphi \le k(\delta(S)) \text{ for all $st$-cuts} \\ & \Leftrightarrow \;\;\; \varphi \le \min_S \Big\{k(\delta(S)): \delta(S) \text{ is a $st$-cut}\Big\} \end{aligned} \] holds for all net flow values \(\varphi\).

6.17

6.18 Strong Duality: Max-Flow Min-Cut Theorem

Consider flow \(x\) of net value \(\varphi\) and \(st\)-cut \(\delta(S)\) of capacity \(k(\delta(S)).\)

Lemma 6.2 If \(\varphi = k(\delta(S))\), then \(x\) is a maximum flow and \(\delta(S)\) is a minimum cut.

Theorem 6.2 The value of a maximum flow is always equal to the capacity of a minimum \(st\)-cut.

We’ll construct an algorithmic proof: reveals and constructs the flow and \(st\)-cut while solving the problem.

6.19 The Ford-Fulkerson Algorithm: Idea

Idea: Starting from given flow \(x\), find an \(st\)-path along which \(\varphi\) can be increased.

6.20 Another Path with Improved Flow

6.21 Forward and Backward Arcs

We can increase flow using:

  • the original arcs in the graph (forward arcs), and

  • arcs whose directions have been flipped (backward arcs).

6.22 Saturated and Empty Arcs

Definition 6.6 An arc \((i,j)\) is saturated if \(x_{ij} = k_{ij}\).

Definition 6.7 An arc \((i,j)\) is empty if \(x_{ij} = 0\).

Observation:

  • If \((i,j)\) is not saturated \((x_{ij} < k_{ij})\) then \(x_{ij}\) can be increased.

  • If \((i,j)\) is not empty \((x_{ij} > 0)\) then \(x_{ij}\) can be decreased.

6.23 Example

6.24 Augmenting Paths

Definition 6.8 (Augmenting Paths) We call a \(st\)-path using a combination of forward and backward arcs an augmenting path if:

  • All forward arcs are not saturated \((x_{ij} < k_{ij})\); and

  • All backward arcs are not empty \((x_{ij} > 0)\).

Idea: we can improve net flow value by increasing flow on forward arcs and decreasing flow on backward arcs.

6.25 Example of an Augmenting Path

6.26 The Auxiliary Network

We can construct an auxiliary network \(G' = (V, A')\) with auxiliary capacity function \(k': A \mapsto \mathbf{R}_+\) that accounts for all possible flow variations with respect to current flow \(x\).

For all \((i,j) \in A\):

  • If \(x_{ij} < k_{ij}\) (not saturated), we add forward arc \((i,j)\) to \(A'\) with \(k'_{ij} = k_{ij} - x_{ij}\).

    • This arc models the possibility of increasing the flow on \((i,j)\).

6.27 The Auxiliary Network – 2

For all \((i,j) \in A\):

  • If \(x_{ij} > 0\) (not empty), we add backward arc \((j,i)\) to \(A'\) with \(k'_{ji} = x_{ij}\).

    • This models the possibility of decreasing the flow on \((i,j)\).

  • If \(0 < x_{ij}< k_{ij}\) we add both a forward arc \((i,j)\) and a backward arc \((j,i)\) to \(A'\).

6.28 The Augmentation Step

Start with flow \(x\) and corresponding auxiliary network \(G'\) with auxiliary capacities \(k'\).

  1. Identify \(st\)-path \(P\) in \(G'\).
  2. Let \(\displaystyle\Delta = \min_{(i,j)\in P} k'_{ij}\).
  3. Add \(\Delta\) to all forward arcs.
  4. Subtract \(\Delta\) from all backward arcs.
